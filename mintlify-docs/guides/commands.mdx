<Note>
**Hammer is a fork of GSD-2.** Slash commands are documented as `/hammer …` (canonical), but every command is also reachable as `/gsd …` (legacy alias). Both forms dispatch to the same handler — pick whichever you have in muscle memory. CLI binaries (`gsd`, `gsd-cli`), the npm package (`gsd-pi`), and filesystem paths (`.gsd/`) keep their legacy names for compatibility.
</Note>

## Extensions

| Command | Description |
|---------|-------------|
| `/hammer extensions list` | List all extensions and their status. User-installed entries show `[user]` plus the install source |
| `/hammer extensions enable <id>` | Enable a disabled extension |
| `/hammer extensions disable <id>` | Disable an extension |
| `/hammer extensions info <id>` | Show extension details |
| `/hammer extensions install <spec>` | Install a user extension. `<spec>` is an npm package, a git URL, or a local path. Restart Hammer to activate. (v2.78) |
| `/hammer extensions uninstall <id>` | Remove a user-installed extension. Warns if other extensions depend on it. (v2.78) |
| `/hammer extensions update [id]` | Update a single user-installed npm extension to its latest version, or all of them when `id` is omitted. Git/local installs are skipped — reinstall to update. (v2.78) |
| `/hammer extensions validate <path>` | Validate an extension package directory against the manifest schema before publishing or installing. (v2.78) |

Install sources are auto-detected: a `git`/`http(s)` URL or `.git` suffix triggers `git clone`; a path that exists on disk is copied locally; everything else is fetched via `npm pack`. Installed extensions land in `~/.gsd/extensions/<id>/` and the registry records the source so `update` can re-fetch.

## MCP tool aliases

When Hammer is run as an MCP server (or invoked from an external MCP client), every tool is dispatched under both a Hammer-canonical name and a GSD legacy alias. Both names route to the same handler — they are not two implementations.

| Canonical (Hammer) | Legacy alias (GSD) | Purpose |
|--------------------|--------------------|---------|
| `hammer_complete_task` | `gsd_complete_task` | Record a completed task |
| `hammer_complete_slice` | `gsd_complete_slice` | Record a completed slice |
| `hammer_complete_milestone` | `gsd_complete_milestone` | Seal a milestone |
| `hammer_validate_milestone` | `gsd_validate_milestone` | Run the validation gate |
| `hammer_plan_milestone` | `gsd_plan_milestone` | Plan a milestone (DB write + roadmap render) |
| `hammer_plan_slice` | `gsd_plan_slice` | Plan a slice (DB write + PLAN render) |
| `hammer_plan_task` | `gsd_plan_task` | Plan a task |
| `hammer_replan_slice` | `gsd_replan_slice` | Replan a slice with structural enforcement |
| `hammer_reassess_roadmap` | `gsd_reassess_roadmap` | Reassess the roadmap |
| `hammer_decision_save` | `gsd_decision_save` | Append a decision to `DECISIONS.md` |
| `hammer_requirement_save` | `gsd_requirement_save` | Append a requirement to `REQUIREMENTS.md` |
| `hammer_save_gate_result` | `gsd_save_gate_result` | Save a quality gate verdict |
| `hammer_journal_query` | `gsd_journal_query` | Query the event journal |
| `hammer_question_round_spiral` | `gsd_question_round_spiral` | Per-round Omega synthesis for question generation |

The full alias table is documented in `packages/mcp-server/src/server.ts`. New tools are registered under both names by default — there is no version where only one form works.

## Headless mode

`gsd headless` runs commands without a TUI — designed for CI, cron jobs, and scripted automation. Headless auto-mode pairs with Hammer's recover-and-resume layer to keep running across crashes.

```bash
gsd headless              # run auto mode
gsd headless next         # run a single unit
gsd headless query        # instant JSON snapshot (~50ms, no LLM)
gsd headless --timeout 600000 auto   # with timeout
gsd headless new-milestone --context brief.md --auto
```

| Flag | Description |
|------|-------------|
| `--timeout N` | Overall timeout in milliseconds (default: 300000) |
| `--max-restarts N` | Auto-restart on crash (default: 3, set 0 to disable) |
| `--json` | Stream events as JSONL to stdout |
| `--model ID` | Override the model |
| `--context <file>` | Context file for `new-milestone` (use `-` for stdin) |
| `--auto` | Chain into auto-mode after milestone creation |

**Exit codes:** `0` = complete, `1` = error/timeout, `2` = blocked.

### `gsd headless query`

Returns a JSON snapshot of the project state — no LLM session, instant response.

```bash
gsd headless query | jq '.state.phase'      # "executing"
gsd headless query | jq '.next'              # next dispatch action
gsd headless query | jq '.cost.total'        # total spend
```

## MCP server mode

```bash
gsd --mode mcp
```

Runs Hammer as a Model Context Protocol server over stdin/stdout, exposing all tools to external AI clients (Claude Desktop, VS Code Copilot, etc.). Every tool is registered under both `hammer_*` (canonical) and `gsd_*` (legacy alias) — see the [MCP tool aliases](#mcp-tool-aliases) table above.
